ses-led-control

Turn the Identify (locate) and Fault LEDs of disk slots on and off, on any Linux box with an SES enclosure — built with TrueNAS SCALE in mind.

Repository Releases Documentation

It walks every SES enclosure on the host, maps each disk to its physical slot by SAS address, and shows the slot status together with the IDENT and FAULT LED state — including empty slots. Pick a slot to toggle its locate LED, or drive it from a one-shot command.

root@nas: ~ — ses-led-control (interactive demo) 

    

    

      >
      
    

  

  
Browser simulation — no hardware is touched. Try
  0, f1, x, r, or click an
  ON/off cell.


  
What it does

  
    
    
  • Enumerates SES enclosures with lsscsi -g.
    • 
    
  • Maps disks to slots by SAS address, via the enclosure
      Additional Element Status (aes) page and each disk's
      sas_address in sysfs.
    • 
    
  • Reads slot status and LED state from the Enclosure Status
      (es) page.
    • 
    
  • Switches LEDs with
      sg_ses --index=<type>,<idx> --set|--clear=ident|fault.
    • 
    
  • Handles both Array device slot and Device slot
      element types automatically.
    • 
  

  
It never writes to disks and does nothing destructive beyond changing an
  LED. LED control depends entirely on the backplane's SES support — if the
  hardware can't do it, neither can this.


  
IDENT vs. FAULT

  

    

      
IDENT (locate)

      
The LED you switch on to physically find a drive — "which bay is
      sdf?". Usually blue or white, steady or blinking.

    

    

      
FAULT

      
Marks a slot as failed. Sometimes driven automatically (e.g. a ZFS
      failure); you can also set it by hand to flag a disk for replacement.
      Typically red or amber.

    

  


  
Requirements

  
    
    
  • root to switch LEDs (list,
      --help, --version run unprivileged)
    • 
    
  • sg3-utils (sg_ses), util-linux
      (lsblk), lsscsi, awk
    • 
  


  
Run

  
One self-contained script — no install, no clone. Download just the script
  into any folder and run it with bash. Works on TrueNAS SCALE,
  where /usr is read-only.

  
sudo apt-get install -y sg3-utils util-linux lsscsi

curl -fsSLO https://raw.githubusercontent.com/nmuellerdo/ses-led-control/main/bin/ses-led-control

sudo bash ses-led-control
bash ses-led-control list
sudo bash ses-led-control ident on sg3:7
bash ses-led-control --dry-run fault on /dev/sdf

  
Line by line: install the dependencies (Debian / Ubuntu / TrueNAS SCALE),
  download just the script, start the interactive menu, print the table (no root
  needed), switch one IDENT LED, and preview a command with
  --dry-run without touching hardware. The blocks contain no
  # comments on purpose — the TrueNAS root shell (zsh) would pass
  them along as arguments when pasted.


  
Targets

  
Every switching command takes a target, which can be a block device
  (/dev/sdf or sdf), a disk serial number, or an
  enclosure slot (sgN:slot, e.g. sg3:7).
