ses-led-control

Turn the Identify (locate) and Fault LEDs of disk slots on and off, on any Linux box with an SES enclosure — built with TrueNAS SCALE in mind.

Repository Releases Documentation

It walks every SES enclosure on the host, maps each disk to its physical slot by SAS address, and shows the slot status together with the IDENT and FAULT LED state — including empty slots. Pick a slot to toggle its locate LED, or drive it from a one-shot command.

root@nas: ~ — ses-led-control 
#   ENCL:SLOT   STATUS       DEV      SERIAL             MODEL                SIZE     FW       IDENT  FAULT
----------------------------------------------------------------------------------------------------------------
0   sg3:0       populated    sda      WDC-SN-0001        WD40EFRX             3.6T     82.0     ON     off
1   sg3:7       populated    sdb      ZAB1234            ST8000               7.3T     SN02     off    ON
2   sg5:2       empty        -        -                  -                    -        -        off    off

number = toggle IDENT | f<n> = toggle FAULT | x = all LEDs off | r = reload | q = quit
>

What it does

  • Enumerates SES enclosures with lsscsi -g.
  • Maps disks to slots by SAS address, via the enclosure Additional Element Status (aes) page and each disk's sas_address in sysfs.
  • Reads slot status and LED state from the Enclosure Status (es) page.
  • Switches LEDs with sg_ses --index=<type>,<idx> --set|--clear=ident|fault.
  • Handles both Array device slot and Device slot element types automatically.

It never writes to disks and does nothing destructive beyond changing an LED. LED control depends entirely on the backplane's SES support — if the hardware can't do it, neither can this.

IDENT vs. FAULT

IDENT (locate)

The LED you switch on to physically find a drive — "which bay is sdf?". Usually blue or white, steady or blinking.

FAULT

Marks a slot as failed. Sometimes driven automatically (e.g. a ZFS failure); you can also set it by hand to flag a disk for replacement. Typically red or amber.

Requirements

  • root to switch LEDs (list, --help, --version run unprivileged)
  • sg3-utils (sg_ses), util-linux (lsblk), lsscsi, awk

Install

# dependencies (Debian / Ubuntu / TrueNAS SCALE)
sudo apt-get install -y sg3-utils util-linux lsscsi

# clone and install to /usr/local
git clone https://github.com/nmuellerdo/ses-led-control.git
cd ses-led-control
sudo ./install.sh

# then, as root:
sudo ses-led-control            # interactive menu
ses-led-control list            # print the table (no root needed to try)
sudo ses-led-control ident on sg3:7
ses-led-control --dry-run fault on /dev/sdf   # preview only, no hardware touched

Prefer a single file? Grab bin/ses-led-control directly and chmod +x it.

Targets

Every switching command takes a target, which can be a block device (/dev/sdf or sdf), a disk serial number, or an enclosure slot (sgN:slot, e.g. sg3:7).
